# 헤르메스 개발지시 프롬프트팩 — BLRO Company Operating OS

## 1. 전체 개발 목표
베를로 회사 운영을 위한 Multi-Persona 기반 Agentic OS MVP를 개발한다. 사용자는 대표처럼 자연어로 지시하고, 시스템은 고객/프로젝트/메일/일정/견적/기술검토/승인/로그를 연결해 실제 업무 실행 초안까지 만든다.

## 2. 반드시 지켜야 할 원칙
1. 모든 데이터는 고객사와 프로젝트에 연결한다.
2. AI가 추정한 내용은 근거 또는 원문을 남긴다.
3. 불확실한 내용은 `확인 필요`로 표시한다.
4. 메일 요약, 초안, 체크리스트, 내부 정리는 자동 처리한다.
5. 메일 발송, 외부 공유, 삭제, 견적 발송은 승인 대기 상태로 보낸다.
6. 모든 실행은 ExecutionRun과 RunStep에 로그를 남긴다.
7. UI는 shadcn/ui 기반으로 작성한다.
8. page.tsx는 얇게 유지하고 feature/server 레이어로 분리한다.

## 3. 개발 시작 프롬프트
```text
첨부 문서 00~12번을 기준으로 BLRO Company Operating OS MVP를 개발해라.

1차 목표:
1. Next.js App Router + TypeScript + Tailwind + shadcn/ui 기본 프로젝트 구조 정리
2. Prisma schema 적용
3. 공통 AppShell, Sidebar, Header, PageHeader, DataTable, Badge 컴포넌트 구현
4. Customers, Projects 기본 CRUD 구현
5. Mail Intelligence 화면 placeholder와 API 구조 구현
6. Presales Review 화면 placeholder와 API 구조 구현
7. Command Center 기본 입력/계획/결과 UI 구현
8. Approvals와 Runs/Logs 기본 구조 구현

외부 연동은 mock adapter로 먼저 구현하고, Gmail/Outlook OAuth는 후속 단계로 분리한다.
외부 발송/삭제/견적 발송은 반드시 ApprovalItem 생성만 하고 실제 실행하지 않는다.
```

## 4. Mail Intelligence 구현 지시
```text
메일 목록 조회, 원문 Preview, 고객명/담당자/요청사항/제품군 추출, 타입 분류, 답변 필요 여부 판단, 긴급도 계산, 프로젝트 연결, 답장 초안 생성을 구현해라. 실제 발송은 ApprovalItem 생성으로 처리한다.
```

## 5. Presales Review 구현 지시
```text
고객 요청 원문 입력, 제품군 선택, 제품군별 체크리스트 생성, 부족 정보 감지, 가능/불가/확인필요 판단, 고객 회신 초안, 벤더/파트너 문의 초안, 프로젝트 저장을 구현해라.
```

## 6. Approval Gateway 구현 지시
```text
SEND_EMAIL, SEND_MESSAGE, SEND_QUOTE, EXTERNAL_SHARE, DELETE, MOVE, PUBLISH, COST_ACTION은 직접 실행하지 말고 ApprovalItem을 생성해라. 승인 전에는 절대 실제 외부 발송하지 않는다.
```
