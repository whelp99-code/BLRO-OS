# BLRO Company Operating OS Multi-Persona User Story

## 1. 문서 목적

이 문서는 베를로 회사 운영 자동화 시스템을 설계하기 위한 핵심 페르소나와 사용자 스토리를 정의한다.  
기존 Agentic OS MVP 템플릿의 화면명세서, 와이어프레임, 폴더구조, Prisma 스키마, 컴포넌트 분해표, 개발 태스크 체크리스트가 이 문서를 기준으로 파생되어야 한다.

## 2. 운영 목표

베를로의 영업, 프리세일즈, 기술지원, 프로젝트 관리, 견적, 메일 대응, 파트너 협업, 재무 관리를 하나의 AI 기반 운영 체계로 통합한다.

최종 목표는 다음과 같다.

> 대표가 모든 일을 직접 기억하고 지시하지 않아도, AI가 고객 메일·일정·프로젝트·견적·기술검토·비용 정보를 분석해서 오늘 해야 할 일, 위험한 일, 돈 되는 일, 놓치면 안 되는 일을 자동으로 정리하고 실행 초안까지 만든다.

## 3. 핵심 운영 객체

```text
Customer
 └── Project
      └── Request
           └── Action Item
                └── Owner / Due Date / Status / Artifact / Approval
```

모든 화면, API, 데이터 모델, 에이전트 동작은 위 구조를 기준으로 연결한다.

# 4. Persona 1 — 대표 / 의사결정자

## 역할
- 회사 전체 운영 판단
- 고객/파트너/벤더 우선순위 결정
- 매출 가능 프로젝트 선별
- 승인 필요 작업 최종 확인
- 비용과 리스크 관리

## 주요 문제
- 고객 메일, 일정, 견적, 기술검토 요청이 흩어져 있다.
- 어떤 일을 먼저 처리해야 하는지 매번 직접 판단해야 한다.
- 프로젝트별 진행 상태와 후속 액션을 기억에 의존한다.
- 매출 가능성이 높은 건과 단순 대응 건이 섞여 있다.
- 승인해야 할 작업과 자동 처리 가능한 작업이 구분되지 않으면 위험하다.

## User Story
대표로서, 나는 매일 회사의 전체 상황을 한눈에 보고 싶다.  
그래서 오늘 반드시 처리해야 할 고객 요청, 매출 가능성이 높은 프로젝트, 지연 업무, 승인 필요 작업을 빠르게 판단하고 싶다.

## Acceptance Criteria
- 매일 아침 고객/프로젝트/메일/일정 기준 업무 브리핑을 제공한다.
- 긴급도, 매출 가능성, 마감일 기준으로 업무 우선순위를 표시한다.
- 대표 승인 필요 항목을 별도 영역으로 분리한다.
- 외부 발송, 최종 견적, 파일 삭제, 외부 공유는 승인 후 실행한다.
- 단순 요약, 초안, 체크리스트, 내부 정리는 자동 수행한다.

## 주요 화면 반영
- Dashboard
- Approvals
- Finance
- Command Center
- Projects

## 핵심 위젯
| 위젯 | 설명 |
|---|---|
| 오늘의 핵심 업무 | 오늘 반드시 처리해야 할 업무 |
| 긴급 고객 요청 | 장애, 견적 마감, 미응답 고객 |
| 매출 가능 프로젝트 | 수주 가능성이 높은 건 |
| 승인 필요 | 발송/공유/견적/삭제 전 승인 |
| 비용 현황 | 월 고정비, 구독비, 예산 초과 |

# 5. Persona 2 — 영업 담당자

## 역할
- 신규 고객 발굴
- 기존 고객 후속 관리
- 파트너 공동 영업
- 제품 제안 기회 발굴
- 고객별 제안 이력 관리

## 주요 문제
- 기존 고객에게 어떤 제품을 추가 제안할지 판단이 어렵다.
- 고객 규모, 산업, 기존 인프라 정보가 정리되지 않는다.
- 후속 연락 타이밍을 놓친다.
- 고객별 히스토리가 메일, 회의록, 메신저에 흩어져 있다.

## User Story
영업 담당자로서, 나는 고객별로 제안 가능한 제품과 다음 액션을 자동으로 추천받고 싶다.  
그래서 기존 고객에게 AP, 방화벽, 스위치, HCI, DR, SASE 같은 제품을 효율적으로 제안하고 싶다.

## Acceptance Criteria
- 고객별 기존 도입 제품, 사용 회선, 인프라 규모를 저장한다.
- 고객 규모에 따라 제안 가능한 제품군을 추천한다.
- 50명 이하 고객, 중견 고객, 엔터프라이즈 고객을 구분한다.
- 후속 연락 필요 고객을 자동 표시한다.
- 고객별 메일 초안, 제안 요약, 미팅 아젠다를 자동 생성한다.

## 제안 매핑 예시
| 고객 상태 | 추천 제품 | 이유 |
|---|---|---|
| 회선 사용 중 | AP / Switch / Firewall | 네트워크 확장 제안 가능 |
| 서버 3대 보유 | HCI Lite / 서버 가상화 | 소규모 가상화 전환 가능성 |
| 해외지사 있음 | SASE / VPN / ZTNA | 해외 접속 안정성 개선 |
| 백업 미흡 | DR / Backup 연동 | 장애 대응 제안 가능 |

## 주요 화면 반영
- Customers
- Projects
- Proposal / Quote Desk
- Dashboard
- Command Center

# 6. Persona 3 — 프리세일즈 / 기술 컨설턴트

## 역할
- 고객 요구사항 분석
- 기술 가능 여부 검토
- 제품 매핑
- 구성안 작성
- PoC 설계
- 고객 회신용 기술 초안 작성

## 주요 문제
- 고객이 제공한 정보가 불완전하다.
- CPU, 메모리, 디스크, 네트워크 정보가 자주 누락된다.
- 제품별 기능 가능 여부 확인이 반복된다.
- 기술 검토 메일을 매번 새로 작성해야 한다.
- 제안서와 실제 구성 간 불일치 위험이 있다.

## User Story
프리세일즈 담당자로서, 나는 고객 요청을 입력하면 필요한 추가 정보, 가능 여부, 권장 구성, 회신 초안을 자동으로 받고 싶다.  
그래서 기술 검토 시간을 줄이고 제안 품질을 높이고 싶다.

## Acceptance Criteria
- 고객 요청에서 부족한 정보를 자동 추출한다.
- HCI, DR, SASE, NGAF, IAG, SCP, SKE 등 제품군별 확인 항목을 제시한다.
- CPU, Memory, Disk, Network, VM 수량, RTO/RPO, 회선 구성 등을 체크리스트로 만든다.
- 고객 회신 메일 초안은 간단하고 실무적인 형식으로 작성한다.
- 가능/불가/확인 필요를 구분한다.

## 기술 검토 체크리스트 예시
| 확인 항목 | 필요 여부 | 설명 |
|---|---:|---|
| 노드 수 | 필수 | 총 가용 리소스 산정 |
| 노드별 CPU Core | 필수 | vCPU 오버커밋 계산 |
| 노드별 Memory | 필수 | VM 수량 산정 핵심 |
| VM당 vCPU/RAM | 필수 | 사용자 프로파일별 산정 |
| HA 여유율 | 필수 | N+1 기준 여부 확인 |
| 스토리지 IOPS | 권장 | 부팅 폭주/업무 성능 확인 |

## 주요 화면 반영
- Presales Review
- Proposal / Quote Desk
- Projects
- Mail Intelligence
- Command Center

# 7. Persona 4 — 기술지원 / 엔지니어

## 역할
- PoC 수행
- 구축 작업
- 장애 대응
- 로그/설정 분석
- 작업내역서 작성
- 결과보고서 작성

## 주요 문제
- 고객 환경 정보가 불완전하다.
- 장애 원인 분석 시 캡처, 로그, 설정값이 흩어져 있다.
- 반복되는 기술 문의가 많다.
- 구축 후 작업내역서 작성이 번거롭다.
- PoC 결과 정리가 늦어진다.

## User Story
기술지원 담당자로서, 나는 고객 환경 정보와 작업 내용을 기반으로 작업계획서, 장애 분석, PoC 결과 보고서를 자동으로 만들고 싶다.  
그래서 기술 대응 속도와 고객 보고 품질을 높이고 싶다.

## Acceptance Criteria
- 고객별 장비, IP, VLAN, 회선, VM, 라이선스 정보를 구조화한다.
- 작업 전 체크리스트를 자동 생성한다.
- 작업 중 발생한 이슈를 원인/조치/결과로 정리한다.
- PoC 종료 후 결과보고서 초안을 자동 작성한다.
- 미확인 항목은 확인 필요로 남긴다.

## 작업내역 구조 예시
| 구분 | 내용 |
|---|---|
| 고객사 | 고객사명 |
| 프로젝트 | PoC / 구축 / 장애 대응 |
| 대상 | VM / 네트워크 / 보안 장비 / 회선 |
| 핵심 확인 | RTO, RPO, 동기화, VPN, VLAN, 성능 |
| 산출물 | 작업계획서, 체크리스트, 결과보고서 |

## 주요 화면 반영
- Projects
- Presales Review
- Runs / Logs
- Artifacts
- Command Center

# 8. Persona 5 — 운영 관리자 / PM

## 역할
- 프로젝트 상태 관리
- 담당자/마감일 관리
- 고객 요청과 내부 액션 연결
- 지연 업무 추적
- 산출물 상태 관리

## 주요 문제
- 프로젝트별 진행상태가 명확하지 않다.
- 누가 무엇을 해야 하는지 매번 확인해야 한다.
- 고객 요청과 내부 업무가 연결되지 않는다.
- 일정 지연이나 미응답 메일을 놓칠 수 있다.

## User Story
운영 관리자로서, 나는 모든 고객 프로젝트의 상태를 자동으로 추적하고 싶다.  
그래서 지연 업무, 담당자 미배정, 고객 회신 대기, 견적 대기 상태를 한눈에 확인하고 싶다.

## Acceptance Criteria
- 프로젝트 상태를 자동 분류한다.
- 신규 문의, 정보요청, 견적작성, 기술검토, 제안진행, PoC, 계약, 완료 상태를 관리한다.
- 각 프로젝트별 다음 액션을 표시한다.
- 지연된 업무는 경고 표시한다.
- 회의록, 메일, 첨부파일을 프로젝트에 연결한다.

## 프로젝트 상태값
| 상태 | 의미 |
|---|---|
| New Lead | 신규 문의 |
| Qualification | 고객 요구사항 확인 중 |
| Tech Review | 기술 검토 중 |
| Proposal | 제안/견적 작성 중 |
| PoC | PoC 진행 중 |
| Negotiation | 가격/계약 협의 중 |
| Won | 수주 |
| Lost | 실패 |
| Hold | 보류 |

## 주요 화면 반영
- Projects
- Dashboard
- Workflows
- Runs / Logs
- Approvals

# 9. Persona 6 — 재무 / 경영관리

## 역할
- 예상 매출 관리
- 확정 매출 관리
- 비용 관리
- 구독비 관리
- 프로젝트별 수익성 관리
- 미수/리스크 확인

## 주요 문제
- AI 툴, SaaS, 개발 도구 구독비가 늘어난다.
- 프로젝트별 수익성을 따로 계산하기 어렵다.
- 매출 가능성과 실제 입금 상태가 분리되어 있다.
- 대표 급여, 직원 급여, 차량비, 세금 관련 판단 자료가 필요하다.

## User Story
재무 담당자로서, 나는 매출 예정, 확정 매출, 비용, 순이익, 구독비를 자동으로 정리받고 싶다.  
그래서 회사의 현금흐름과 비용 구조를 빠르게 판단하고 싶다.

## Acceptance Criteria
- 월별 매출/비용/순이익을 표시한다.
- 고객별 예상 매출과 확정 매출을 구분한다.
- 구독비, 인건비, 차량비, 외주비를 분리한다.
- 월 예산 초과 가능성을 경고한다.
- 프로젝트별 수익성을 계산한다.

## 관리 항목
| 항목 | 관리 기준 |
|---|---|
| 확정 매출 | 계약/발주 기준 |
| 예상 매출 | 제안/견적 단계 기준 |
| 고정비 | 구독비, 인건비, 차량비 |
| 변동비 | 외주비, 프로젝트성 비용 |
| 순이익 | 매출 - 비용 |
| 리스크 | 미수금, 일정 지연, 비용 초과 |

## 주요 화면 반영
- Finance
- Dashboard
- Projects
- Proposal / Quote Desk

# 10. Persona 7 — 고객사 담당자

## 역할
- 제품 도입 검토
- 기술 질의
- 견적 요청
- PoC 요청
- 내부 보고

## 주요 문제
- 어떤 정보를 제공해야 하는지 모른다.
- 기술 검토 요청 후 답변을 기다려야 한다.
- 견적/제안서 기준이 명확하지 않다.
- 내부 보고용 요약자료가 필요하다.

## User Story
고객사 담당자로서, 나는 제품 검토에 필요한 요청사항과 준비자료를 명확하게 안내받고 싶다.  
그래서 내부 검토와 의사결정을 빠르게 진행하고 싶다.

## Acceptance Criteria
- 고객 요청에 따라 필요한 준비자료 목록을 안내한다.
- 기술 가능 여부를 간단히 설명한다.
- 내부 보고용 요약본을 제공한다.
- 견적 산정을 위한 필수 정보를 요청한다.
- 불확실한 내용은 확인 필요로 분리한다.

## 고객 요청자료 템플릿
| 구분 | 요청 정보 |
|---|---|
| 서버 | 제조사, 모델, CPU, Memory, Disk |
| 가상화 | VMware, Hyper-V, Nutanix, 물리서버 여부 |
| VM | VM명, OS, vCPU, Memory, Disk |
| 네트워크 | VLAN, IP 대역, 방화벽, VPN |
| 백업/DR | RTO, RPO, 백업 방식 |
| 일정 | PoC 희망일, 구축 희망일 |

## 주요 화면 반영
- Mail Intelligence
- Presales Review
- Proposal / Quote Desk
- Artifacts

# 11. Persona 8 — 파트너 / 벤더

## 역할
- 제품 공급
- 견적 지원
- 기술 확인
- 라이선스 확인
- 공동 영업 지원

## 주요 문제
- 고객 요구사항이 정리되지 않은 상태로 전달된다.
- 견적 요청 시 필요한 정보가 누락된다.
- 제품별 담당자와 기술 확인 루트가 다르다.
- 공동 영업 진행상태가 불명확하다.

## User Story
파트너 담당자로서, 나는 고객 요구사항과 필요한 견적 정보를 정리된 형태로 받고 싶다.  
그래서 빠르게 견적과 기술 검토를 지원하고 싶다.

## Acceptance Criteria
- 고객 요청사항을 제품별로 정리한다.
- 견적 산정에 필요한 수량, 라이선스, 기간, 옵션을 분리한다.
- 벤더 확인 필요 항목을 별도로 표시한다.
- 공동 영업 상태를 추적한다.
- 파트너별 진행 현황을 관리한다.

## 주요 화면 반영
- Proposal / Quote Desk
- Partner Requests
- Projects
- Approvals

# 12. 공통 자동화 User Story

## 12.1 메일 분석
사용자로서, 나는 수신 메일을 AI가 자동으로 분석해서 고객 요청, 긴급도, 필요한 액션을 정리해주길 원한다.

### Acceptance Criteria
- 메일 본문에서 고객명, 제품명, 요청사항을 추출한다.
- 답변 필요 메일만 우선순위로 표시한다.
- 고객에게 보낼 답장 초안을 생성한다.
- 메일 전송은 반드시 승인 후 진행한다.

## 12.2 프로젝트 자동 생성
사용자로서, 나는 고객 문의가 들어오면 프로젝트 후보가 자동 생성되길 원한다.

### Acceptance Criteria
- 신규 고객 문의는 자동으로 프로젝트 후보로 생성된다.
- 기존 고객이면 기존 프로젝트와 연결한다.
- 중복 프로젝트는 생성하지 않는다.
- 프로젝트명은 고객명 + 제품군 + 목적 기준으로 생성한다.

## 12.3 견적/제안 지원
사용자로서, 나는 고객 요청을 기반으로 필요한 견적 항목과 제안 구성을 자동으로 받고 싶다.

### Acceptance Criteria
- 견적 산정에 필요한 필수 정보가 누락되면 표시한다.
- 제품별 수량 산정 기준을 명확히 표시한다.
- 고객에게 보낼 정보 요청 메일을 생성한다.
- 최종 견적 금액 입력 전에는 확정 견적으로 표시하지 않는다.

## 12.4 기술 검토 자동화
사용자로서, 나는 고객 기술 문의를 입력하면 가능 여부, 확인 필요 항목, 회신 초안을 자동으로 받고 싶다.

### Acceptance Criteria
- 명확한 기능은 가능/불가로 표시한다.
- 불확실한 기능은 확인 필요로 표시한다.
- 고객 회신은 과장 없이 작성한다.
- 내부 검토용 상세 내용과 고객 발송용 간단 내용을 분리한다.

## 12.5 일정/할 일 관리
사용자로서, 나는 오늘 해야 할 일과 이번 주 마감 업무를 자동으로 받고 싶다.

### Acceptance Criteria
- 오늘 처리해야 할 업무를 우선순위로 정렬한다.
- 지연된 업무를 별도 표시한다.
- 고객 회신 대기 상태를 관리한다.
- 완료된 업무는 자동으로 히스토리에 기록한다.

# 15. MVP 우선순위

## Phase 1 — 대표 운영 브리핑
- 오늘 일정
- 미응답 메일
- 긴급 고객 요청
- 견적/제안 마감
- 프로젝트 상태
- 승인 필요 작업

## Phase 2 — 메일 기반 프로젝트 관리
- 고객명 추출
- 요청사항 분류
- 프로젝트 연결
- 답장 초안 생성
- 후속 액션 추천

## Phase 3 — 프리세일즈 자동화
- HCI/DR/SASE/보안 제품별 체크리스트
- 누락 정보 자동 감지
- 고객 요청자료 메일 초안
- 기술 검토 요약

## Phase 4 — 견적/제안 자동화
- BOM 초안
- 제품별 수량 산정
- 견적 요청 메일
- 고객 제안 요약
- 파트너 전달용 정리본

## Phase 5 — 경영관리
- 월별 매출 관리
- 예상/확정 매출 분리
- 비용 관리
- 프로젝트별 수익성
- 월 예산 초과 경고
